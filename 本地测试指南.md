# 同事本地测试指南

本文说明拿到本仓库压缩包并解压后，如何在本地安装依赖、启动服务，以及测试时常用地址与账号。业务接口细节见仓库内 **`server/API接口文档.md`**。

---

## 1. 环境要求

| 项 | 说明 |
|----|------|
| **操作系统** | Windows / macOS / Linux 均可 |
| **Node.js** | **建议 22.5 及以上**（后端 `server` 声明 `>=22.5.0`，且使用 Node 内置 SQLite；版本过低可能无法启动） |
| **包管理** | **npm**（与项目当前用法一致；也可用 pnpm/yarn，但本文以 npm 为准） |

安装 Node 后，在终端执行以下命令确认版本：

```bash
node -v
npm -v
```

---

## 2. 解压与进入目录

1. 将收到的压缩包解压到任意路径（路径中尽量避免仅个别工具不支持的极端特殊字符）。
2. 打开终端（Windows 可用 PowerShell 或 CMD），**进入解压后的项目根目录**（根目录内应能看到 `package.json`、`server`、`client` 等文件夹）。

示例（请把路径改成你本机实际路径）：

```bash
cd "E:\新能源材料进销系统\project-47_ea974478_a-1776656089175-rb9xhq1c"
```

---

## 3. 安装依赖

在**项目根目录**执行一次即可（本仓库为 npm workspaces，会安装根目录及 `server`、`client`、`admin`、`inventory-web` 等子包依赖）：

```bash
npm install
```

- 若网络较慢，可配置国内 npm 镜像后再执行。
- 安装结束后若提示 `found 0 vulnerabilities` 或仅有 funding 提示，一般可忽略 funding，不影响运行。

---

## 4. 启动方式（二选一）

**保持该终端窗口不要关闭**；停止服务时在对应终端按 **`Ctrl + C`**。

### 方式 A：进销存业务前端 + 后端（常用）

同时启动 **API 服务** 与 **库房/进销存 Web（Vite）**：

```bash
npm run dev:inventory
```

| 服务 | 默认地址 |
|------|-----------|
| 后端 API | http://localhost:3001 |
| Swagger 接口文档（浏览器） | http://localhost:3001/docs |
| 进销存前端页面 | http://localhost:5173 |

前端通过代理访问 `/api`（详见 `inventory-web/vite.config.js`），一般**只需浏览器打开 5173** 即可联调测试。

### 方式 B：全套开发（后端 + 三个前端）

同时启动 **server、client、admin**：

```bash
npm run dev
```

| 服务 | 默认地址（以当前配置为准） |
|------|---------------------------|
| 后端 API | http://localhost:3001 |
| admin 前端 | http://localhost:5174 |
| client 前端 | http://localhost:5175 |

## 5. 首次运行与数据说明

- **数据库**：SQLite，文件位于 **`server/data/app.sqlite`**。首次成功启动后端后会自动建库、建表。
- **默认管理员**：空库时后端日志会提示自动创建 **`admin` / `admin123`**（角色为统计部 `statistics`，可审核入库、管理用户等）。库房角色为 `warehouse`，负责收货定价、入出库等。
- **自助注册**：可通过接口 `POST /api/register` 注册（默认库房角色）；若需关闭，服务端可设置环境变量 `DISABLE_PUBLIC_REGISTER=1`（测试一般不必设）。

## 6. 端口占用与常见问题

- **3001、5173（或 5174/5175）被占用**：按下面步骤释放端口后重试（以 Windows 为例）。

```powershell
# 1) 查是谁占用了 5173
netstat -ano | findstr :5173

# 2) 记下最后一列 PID，结束进程（示例 PID=12345）
taskkill /PID 12345 /F

# 3) 重新启动
npm run dev:inventory
```

若你不方便关闭占用进程，可临时把前端端口改掉后再启动：

```bash
# 修改 inventory-web/package.json 中 dev 脚本端口（示例改为 5176）
"dev": "vite --port 5176"
```

改端口后前端访问地址也改为 `http://localhost:5176`。
- **页面能开但接口报错**：确认后端终端无报错，并访问 http://localhost:3001/api/health 是否返回正常 JSON。
- **修改了依赖**：在根目录重新执行 `npm install`。
- **Node 版本过低**：升级到 **22.5+** 后再试。

## 7. 可选环境变量（本地测试一般可不设）

在启动后端的终端里设置（Windows PowerShell 示例）：

```powershell
$env:PORT="3001"
$env:JWT_SECRET="你自己设的密钥"
$env:DISABLE_PUBLIC_REGISTER="1"
npm run dev:inventory
```

| 变量 | 作用 |
|------|------|
| `PORT` | 后端监听端口，默认 `3001` |
| `JWT_SECRET` | JWT 签名密钥；不设则使用内置演示密钥（仅本地测试可接受） |
| `DISABLE_PUBLIC_REGISTER` | 设为 `1` 时关闭自助注册 |

若改了 `PORT`，需同步修改各前端 Vite 里对后端的代理目标，否则页面无法正确请求 API。

## 8. 生产构建（一般测试可跳过）

根目录一键构建各子项目：

```bash
npm run build
```

日常功能测试通常只需 **`npm run dev:inventory`** 或 **`npm run dev`** 即可。
