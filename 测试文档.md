# 新能源材料进销存系统 — 测试文档

本文面向**功能测试、验收测试与回归测试**人员，说明测试环境入口、按何种顺序执行业务流程，以及如何记录结果。

**当前测试执行环境（正式服务器）**：测试人员主要在浏览器访问 **[https://redspiderbc.cn/project3/](https://redspiderbc.cn/project3/)** 进行功能与验收测试（与仓库构建约定 `INVENTORY_BASE=/project3/` 一致）。技术接口细节以仓库内 **`server/API接口文档.md`** 为准；在服务器上调试 HTTP 接口时，优先使用下文 **§2.3** 中的 **Swagger** 与健康检查地址。

## 1. 文档与范围说明

| 项 | 说明 |
|----|------|
| **被测主体** | 后端 API（`server/`）+ 进销存业务前端（`inventory-web/`） |
| **主测入口（服务器）** | 浏览器打开 **https://redspiderbc.cn/project3/**（HTTPS、子路径部署） |
| **本地联调（开发自测）** | 仓库根目录 `npm run dev:inventory`，浏览器 **http://localhost:5173**（详见 `本地测试指南.md`） |

| **版本与行为** | 以当前线上部署版本为准；对照代码时以仓库 `server/src`、`inventory-web` 及 `server/openapi.json` 为准 |

### 1.1 测试目标摘要

- **认证与角色**：登录、JWT、统计部（`statistics`）与库房（`warehouse`）权限隔离；默认 `admin` 为超级管理员（前后端均视为同时拥有统计与库房能力）。
- **主数据**：库房 CRUD、品种只读列表。
- **收货定价**：库房录入/修改/删除定价；单价与凭证字段校验。
- **对外统一报价**：统计部发布销售价；库房/统计均可查看。
- **入库**：库房创建待审核入库单（单价须与**当前最新收货定价**一致）；统计部审核通过或驳回；库房仅可删除**待审核**且无出库关联的单据。
- **出库**：库房创建待完成出库（FIFO 扣减）；确认实际出库须上传磅单图；库存不足等业务错误。
- **报表与预警**：入库汇总预警、库房库存报表（含库存状态筛选）。

## 3. 测试账号与角色策略

### 3.1 空库默认账号

| 用户名 | 密码 | 后端角色 | 说明 |
|--------|------|----------|------|
| `admin` | `admin123` | `statistics` | **超级管理员**：JWT 虽为统计部，但 `username=admin` 在前端映射为全权限，且服务端对 admin 同时放行库房类接口 |

### 3.2 建议准备的测试账号（通过接口创建）

前端 **`inventory-web` 无独立「用户管理」菜单**；创建统计部/库房测试账号请使用：

- **Swagger**（服务器上通常为 `https://redspiderbc.cn/project3/docs`）：`POST /api/admin/users`（需已登录统计部 Token），或  
- **自助注册**：`POST /api/register`（固定创建 **`warehouse`** 角色，不可注册为统计部；完整 URL 前缀与 §2.3 一致）

建议至少准备：

| 用途 | 角色 | 创建方式 |
|------|------|----------|
| 纯库房流程 | `warehouse` | `POST /api/register` 或 admin 调 `POST /api/admin/users` 指定 `role` |
| 纯统计流程 | `statistics` | 仅 `admin` 或其它 statistics 用户调 `POST /api/admin/users` |

**环境变量**：若设置 `DISABLE_PUBLIC_REGISTER=1`，则 `POST /api/register` 应返回 **403**，需在测试中验证该开关（如需要）。

## 4. 测试流程总览（推荐阅读顺序）

业务存在**强依赖顺序**，建议测试与演示按下列顺序执行，可减少无效失败（例如未定价则无法建入库单）。

```mermaid
flowchart LR
  A[库房维护] --> B[收货定价]
  B --> C[统计部发布对外报价]
  C --> D[库房入库单]
  D --> E[统计部审核]
  E --> F[库房出库]
  F --> G[报表与预警核对]
```

1. **健康检查**（可选）：浏览器或工具访问 `https://redspiderbc.cn/project3/api/health` → 登录 → 库房列表/品种列表可读  
2. **库房**：新增/编辑/删除（注意删除条件：至少保留一个库房；无关联入出库）  
3. **收货定价**（库房或 admin）：为各测试品种录入**当前生效**定价（含可选凭证）  
4. **对外报价**（统计或 admin）：为品种发布报价（单价 > 0）  
5. **入库单**（库房）：单价必须与**该品种最新收货定价**一致（元/吨，保留两位小数比较）  
6. **审核**（统计）：待审核 → 通过；另测驳回及驳回原因  
7. **出库**（库房）：创建出库 → 完成出库（实际重量、磅单图必填等）  
8. **报表/预警**：与界面数字、列表筛选结果交叉核对

## 5. UI 测试（`inventory-web`）— 按菜单

以下步骤均在浏览器 **https://redspiderbc.cn/project3/** 完成（与 [线上系统入口](https://redspiderbc.cn/project3/) 一致）。对接后端时登录页说明：**仅可使用数据库中已存在用户**；演示账号 `statistics` / `warehouse1` 等为**离线演示**，在 API 模式下已隐藏，勿用作期望账号。

### 5.1 登录与登出

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 打开站点 | 显示登录页 |
| 2 | 空用户名或空密码点登录 | 合理提示（前端校验或接口错误提示） |
| 3 | 错误密码 | 登录失败，401 类提示 |
| 4 | `admin` / `admin123` | 进入主界面，顶部显示用户与角色 |
| 5 | 点击退出 | 回到登录页，本地 Token 清除（再次访问需重新登录） |

### 5.2 菜单权限（与角色对照）

前端根据登录角色**隐藏无权限菜单**（逻辑见 `inventory-web/app.js` 中 `updateMenuVisibility`）：

| 菜单 | 典型可见角色 |
|------|----------------|
| 首页 | 全部 |
| 收货定价 | 库房；统计部也可看（定价页对统计开放只读/协同） |
| 收货入库、出库管理、库房管理 | 库房（admin 全权限） |
| 入库审核、对外报价 | 统计部（admin 全权限） |
| 库存查询、报表中心 | 统计部（`report` 权限） |

**测试要点**：分别用 **纯 warehouse** 与 **纯 statistics** 登录，逐项核对菜单显示是否符合上表；`admin` 应可见全部。

### 5.3 首页（dashboard）

- 核对 **待审核入库** 数量与「入库审核 → 待审核」列表条数一致（允许刷新延迟）。  
- 核对 **库存预警** 与「库存查询 → 预警」数据一致。  
- 查看 **各品种最新收货定价与对外报价** 展示是否与后台数据一致。

### 5.4 库房管理

| 用例 | 步骤 | 期望 |
|------|------|------|
| 新建 | 点击新建库房，填写代码、名称、地址（若表单有），保存 | 列表出现新记录 |
| 编辑 | 修改名称或地址，保存 | 列表更新 |
| 删除 | 删除无业务关联的库房 | 成功；若有关联入出库应失败（见 API 409/错误体） |
| 边界 | 尝试删除最后一个库房 | 应被拒绝（系统至少保留一个库房） |

### 5.5 收货定价

| 用例 | 步骤 | 期望 |
|------|------|------|
| 新增 | 选择品种，填写单价（>0）、录入时间、可选上传/粘贴凭证 | 列表新增；首页/接口 `latest-by-material` 显示最新价 |
| 搜索 | 在搜索框输入品种关键字 | 列表过滤正确 |
| 编辑/删除 | 对已有记录修改或删除（库房权限） | 与接口一致；删除后入库单校验应以**最新价**为准 |

### 5.6 收货入库

| 用例 | 步骤 | 期望 |
|------|------|------|
| 成功创建 | 新建入库单：库房、品种、重量、**单价=当前最新收货定价**、可选照片 | 状态为待审核；统计部在待审核列表可见 |
| 单价不一致 | 故意填写与最新收货定价不同的单价 | 保存失败，错误信息中可能包含服务端返回的 `latestPurchaseUnitPrice` 提示 |
| 删除 | 待审核且无出库关联 | 删除成功 |
| 状态筛选 | 使用状态下拉筛选 | 列表与筛选条件一致 |

### 5.7 入库审核

- **待审核**：打开单据详情，执行「通过」→ 单据进入已审核（业务展示为已审核待出库等语义，与后端 `audit_status` 一致）。  
- **驳回**：填写/不填写驳回原因；驳回后出现在「已驳回」标签。  
- **已审核**：列表只读核对。  
- **关联出库后**：尝试驳回（若业务已产生 FIFO 关联）应被拒绝（见 API 文档说明）。

### 5.8 对外报价

- 使用 **统计部或 admin** 发布某品种的对外报价。  
- 库房账号登录后应能查看报价列表/最新价（只读）。  
- 校验单价、发布时间字段展示。

### 5.9 出库管理

界面含多标签（预出库、实际出库、历史等），测试时建议：

| 用例 | 步骤 | 期望 |
|------|------|------|
| 创建出库 | 选择库房、品种，填写计划出库重量（≤可 FIFO 扣减的库存） | 生成待完成出库单；库存预占合理 |
| 库存不足 | 计划重量超过可用库存 | 创建失败，错误体可能含 `shortfall` 等 |
| 完成出库 | 填写实际重量、上传/填写磅单图 URL（按界面要求） | 状态变为已完成，入库侧状态联动更新 |
| 删除待完成 | 删除 `pending` 出库 | 成功并回滚预占 |

### 5.10 库存查询

- **预警**与**报表**子页：切换标签，核对表格列、分页（如有）、筛选条件。  
- 与 **首页**、**接口报表** 三方抽样核对同一品种、同一库房数据。

### 5.11 报表中心

- 打开各报表入口，检查导出或打印（若有）、日期范围、库房/品种筛选。  
- 空数据、单条数据、大量数据（可造数）下的性能与滚动体验（非功能测试可记为观察项）。

### 5.12 磅单解析（若界面暴露）

- 库房用户上传或输入 OCR 文本后触发解析。  
- 核对解析结果字段与 `server/src/weighbridgeParse.js` 行为一致（**不调用外部 OCR**，仅为规则解析）。

## 6. API 测试（Swagger / 自动化）

### 6.1 通用步骤（服务器）

1. 打开 **https://redspiderbc.cn/project3/docs**（若无法打开见 §2.3 说明）  
2. 调用 `POST /api/admin/login`，请求体与线上一致，复制响应中的 `token`  
3. 点击页面 **Authorize**，输入：`Bearer <token>`（注意 `Bearer` 与 token 之间有一个空格）  
4. 按模块展开接口，使用 **Try it out** 发送请求  

**本地对照**：开发机可打开 `http://localhost:3001/docs`，步骤相同。

### 6.2 建议优先覆盖的接口分组

| 分组 | 方法示例 | 角色 | 重点断言 |
|------|-----------|------|----------|
| 健康 | `GET /api/health` | 无 | 200，`ok` |
| 登录 | `POST /api/admin/login` | 无 | 成功返回 token；失败 401 |
| 用户 | `GET/POST/PUT/DELETE /api/admin/users` | statistics | 409 重名；删除至少保留一人 |
| 注册 | `POST /api/register` | 无 | 201；`DISABLE_PUBLIC_REGISTER` 时 403 |
| 库房 | `GET/POST/PUT/DELETE /api/admin/warehouses` | 已登录 | 删除约束 |
| 品种 | `GET /api/admin/materials` | 已登录 | 列表字段完整 |
| 收货定价 | purchase-prices 系列 | 写操作 warehouse | 单价 >0；batch 批量 |
| 对外报价 | sale-prices 系列 | POST statistics | 无 PUT/DELETE 路由（当前版本） |
| 入库 | inbound-orders 系列 | 创建 warehouse；审核 statistics | 单价与最新定价一致 |
| 出库 | outbound-orders 系列 | warehouse | complete 必填 weighbridge；FIFO |
| 预警/报表 | inbound-summary-alerts、warehouse-stock-report | 已登录 | query 参数组合 |

### 6.3 状态码与错误体（阴性测试）

对每一类资源至少设计：

- **400**：缺字段、非法数值、非法枚举  
- **401**：无 Token、错误 Token、过期 Token（可将系统时间调快或等待 7 天 JWT 过期 — 后者成本高，可与开发确认缩短过期用于测试环境）  
- **403**：正确 Token 但角色不允许（如 statistics 调 `POST /api/admin/purchase-prices`）  
- **404**：不存在的 id  
- **409**：唯一约束（用户名、单号重复）  

错误体一般为：`{ "error": "中文说明" }`，测试记录中应粘贴完整响应。

## 7. 关键业务规则检查表（快速勾选）

测试执行中可按表逐项打勾。

- [ ] 入库单价（元/吨）与**该品种当前最新收货定价**一致（两位小数）  
- [ ] 统计部审核通过后，库房才可按 FIFO 进行出库相关操作  
- [ ] 驳回、删除入库：存在 `outbound_fifo_lines` 关联时不可驳回/不可删除  
- [ ] 出库完成接口：`actualWeight` > 0，磅单图字段必填  
- [ ] 删除 `pending` 出库可回滚预占  
- [ ] 分页参数 `pageSize` 不超过接口上限（多为 100；库存报表见 API 文档）  
- [ ] `admin` 与纯角色账号权限差异符合预期
